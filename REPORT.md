## 2. Описание задачи

### 2.1 Формулировка задачи

**Основная задача**: Построить систему, которая на основе предыдущих оценок фильмов пользователем предсказывает, какие новые фильмы ему понравятся.

**Тип задачи**: Регрессия (предсказание оценки 1-5) + Ранжирование (какие фильмы рекомендовать в первую очередь)

### 2.2 Входные данные

- **Оценки пользователей**: Матрица (пользователь, фильм, оценка)
- **Характеристики фильмов**: Название, жанры, год выпуска, средняя оценка
- **Характеристики пользователей**: История оценок

### 2.3 Выходные данные

- **Рекомендации**: Список из N фильмов с предсказанными оценками
- **Объяснение**: Информация о том, почему рекомендуется фильм

## 4. Разработка модели

### 4.1 Выбор моделей

Была выбрана **гибридная система рекомендаций**, комбинирующая:

#### 1) Коллаборативная фильтрация (User-User)

**Почему выбрана**:
- Хорошо работает с явными оценками пользователей
- Может открывать новые интересы (serendipity)
- Интуитивна: "друзья" помогают выбрать

**Механизм**:
```
1. Вычислить подобие между всеми парами пользователей
   similarity(u1, u2) = cosine_similarity(ratings_u1, ratings_u2)

2. Для каждого пользователя найти K похожих пользователей

3. Рекомендовать фильмы, которые нравились похожим пользователям:
   predicted_rating = average(ratings_similar_users)
```

#### 2) Контентная фильтрация (Content-Based)

**Почему выбрана**:
- Решает cold-start проблему (новые пользователи)
- Интерпретируема: "вам нравился Action, поэтому..."
- Не требует других пользователей

**Механизм**:
```
1. Создать вектор признаков для каждого фильма
   features = [is_action, is_comedy, rating, year, ...]

2. Вычислить подобие между фильмами:
   similarity(m1, m2) = cosine_similarity(features_m1, features_m2)

3. Рекомендовать фильмы, похожие на понравившиеся:
   recommendations = similar_films(liked_movies)
```

#### 3) Гибридный подход

**Комбинация**:
```python
final_score = 0.6 * collab_score + 0.4 * content_score
```

**Преимущества гибридного подхода**:
- Корректирует недостатки обоих методов
- Коллаб. фильтрация находит новые фильмы
- Контентная фильтрация объясняет рекомендации
- Лучше работает при холодном старте

### 7.3 Возникшие трудности и решения

#### Трудность 1: Cold-start проблема
**Проблема**: Новые пользователи не имеют оценок для коллаб. фильтрации

**Решение**: 
- Добавлена контентная фильтрация для новых пользователей
- Рекомендуются популярные фильмы до первых оценок
- После 2+ оценок активируется коллаб. фильтрация

#### Трудность 2: Производительность
**Проблема**: Вычисление подобия для всех пар медленно

**Решение**:
- Использована оптимизация косинусного подобия (sklearn)
- Модели предварительно вычисляются при запуске
- Рекомендации кешируются

#### Трудность 3: Разреженность матрицы
**Проблема**: 93.7% нулей в матрице пользователь-фильм

**Решение**:
- Использована разреженная матрица (scipy.sparse)
- Применена шумоподавление (заполнение нулей средним значением)
- Оптимизирована память

#### Трудность 4: Версионирование моделей
**Проблема**: Как отслеживать различные версии моделей?

**Решение**:
- Сохранение в отдельных файлах
- Метаданные о дате обучения
- Возможность откката к предыдущей версии
